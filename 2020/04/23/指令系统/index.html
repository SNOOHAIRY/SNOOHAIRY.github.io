一.指令系统的发展和性能要求

1.指令系统的发展

(1)指令（机器指令）的定义

是指要求计算机执行某种操作的命令，是计算机硬件能够直接识别和执行的二进制机器代码，又称机器指令。

(2)指令系统

一台计算机的所有的指令结合成为该机的指令系统和指令集。

(3)系统计算机

是指具有相同的基本指令系统和基本体系结构，但具有不同组合和实现的一系列不同型号的机器。

2.一条指令应包含如何信息：

(1)进行何种操作

即操作性质。体现再指令中被成为操作码。

(2)操作的对象

数据来源以及如何寻找操作数。体现在指令中被成为地址码。

(3)操作的结果

指令操作结果存放何处。

(4)下一条指令定位

下一条指令的起始地址，一般由PC指示 只有跳转指令需要

program count (程序计数器 PC=？)

3.对指令系统性能的要求

完备性

指要求指令系统功能完备

有效性

规整性

包括对称性、匀齐性、指令格式和数据格式的一致性。

⭐二.指令格式

1.地址码

按地址码的多少，指令一般可以分为（固定操作码长）：

🟤三地址指令

OP A1 A2 A3

指令功能一： A1 OP A2->A3

🟤二地址指令

OP A1 A2

🟤一地址指令

OP A1

🟤零地址指令

OP
